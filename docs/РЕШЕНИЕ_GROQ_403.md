# Решение проблемы 403 Forbidden с Groq API

## Ситуация:
- ✅ Токен создан на **console.groq.com** (это правильно!)
- ✅ Токен начинается с `gsk_` (формат верный)
- ❌ Получаем 403 Forbidden при запросах

## Возможные причины:

### 1. Аккаунт не активирован
**Решение:**
- Войдите на https://console.groq.com
- Проверьте, что email подтвержден
- Убедитесь, что вы приняли Terms of Service
- Проверьте, нет ли предупреждений в консоли

### 2. Токен создан, но не активирован
**Решение:**
- Удалите текущий токен
- Создайте новый токен
- Обновите `.env` файл
- Подождите 2-3 минуты после создания

### 3. Нужно подтвердить аккаунт
**Решение:**
- Проверьте почту на письмо от Groq
- Проверьте папку спам
- Подтвердите регистрацию, если требуется

### 4. Ограничения по региону (маловероятно)
**Решение:**
- Groq обычно работает без VPN
- Если проблема сохраняется, попробуйте с VPN

## Быстрая проверка:

1. **Зайдите на console.groq.com**
2. **Проверьте статус аккаунта:**
   - Есть ли предупреждения?
   - Подтвержден ли email?
   - Приняты ли Terms of Service?

3. **Проверьте токен:**
   - Убедитесь, что токен активен
   - Проверьте, нет ли ограничений на токене

4. **Создайте новый токен:**
   - Удалите старый
   - Создайте новый
   - Обновите `.env`
   - Попробуйте снова

## Альтернативное решение:

Если проблема сохраняется, можно использовать **GigaChat** - он точно работает в России и имеет бесплатный тариф.

Для переключения на GigaChat:
1. Получите API ключ на https://developers.sber.ru/gigachat
2. Обновите код в `server-local.js` и `api/chat.js`
3. Добавьте `GIGACHAT_API_KEY` в `.env`
