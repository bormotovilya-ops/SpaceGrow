# Инструкция по публикации на GitHub и Vercel

## Шаг 1: Установка Git (если не установлен)

1. Скачайте Git с официального сайта: https://git-scm.com/download/win
2. Установите Git, следуя инструкциям установщика
3. Перезапустите терминал/командную строку

## Шаг 2: Инициализация Git репозитория

Откройте терминал в папке проекта и выполните:

```bash
# Инициализация git репозитория
git init

# Добавление всех файлов
git add .

# Первый коммит
git commit -m "Initial commit: Sales Funnel MiniApp"
```

## Шаг 3: Создание репозитория на GitHub

1. Войдите в свой аккаунт на https://github.com
2. Нажмите кнопку **"+"** в правом верхнем углу → **"New repository"**
3. Заполните:
   - **Repository name**: `sales-funnel-miniapp` (или любое другое название)
   - **Description**: "Архитектор автоматизированных цепочек продаж"
   - Выберите **Public** или **Private**
   - **НЕ** ставьте галочки на "Add a README file", "Add .gitignore", "Choose a license" (у нас уже есть файлы)
4. Нажмите **"Create repository"**

## Шаг 4: Подключение к GitHub

После создания репозитория GitHub покажет инструкции. Выполните в терминале:

```bash
# Замените YOUR_USERNAME и YOUR_REPO_NAME на ваши данные
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# Переименуйте ветку в main (если нужно)
git branch -M main

# Отправьте код на GitHub
git push -u origin main
```

Если GitHub попросит авторизацию:
- Используйте **Personal Access Token** вместо пароля
- Создайте токен: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token
- Выберите scope: `repo` (полный доступ к репозиториям)

## Шаг 5: Деплой на Vercel

### Вариант 1: Через веб-интерфейс (рекомендуется)

1. Войдите в аккаунт на https://vercel.com
2. Нажмите **"Add New..."** → **"Project"**
3. Нажмите **"Import Git Repository"**
4. Выберите ваш репозиторий с GitHub
5. Настройки проекта:
   - **Framework Preset**: Vite (должен определиться автоматически)
   - **Root Directory**: `./` (корень проекта)
   - **Build Command**: `npm run build` (уже указано)
   - **Output Directory**: `dist` (уже указано)
   - **Install Command**: `npm install` (уже указано)
6. Нажмите **"Deploy"**

### Вариант 2: Через Vercel CLI

```bash
# Установите Vercel CLI глобально
npm install -g vercel

# Войдите в аккаунт
vercel login

# Деплой
vercel

# Для продакшена
vercel --prod
```

## Шаг 6: Настройка переменных окружения (если нужно)

Если в будущем понадобятся переменные окружения:
1. В Vercel: Project → Settings → Environment Variables
2. Добавьте нужные переменные
3. Передеплойте проект

## Шаг 7: Обновление кода

После каждого изменения кода:

```bash
# Добавить изменения
git add .

# Создать коммит
git commit -m "Описание изменений"

# Отправить на GitHub
git push

# Vercel автоматически передеплоит проект
```

## Важные замечания

1. **Backend (Python/Flask)**: 
   - Текущий backend не будет работать на Vercel напрямую
   - Если нужен backend, рассмотрите:
     - Vercel Serverless Functions (переписать на Node.js)
     - Отдельный хостинг для Python (Railway, Render, Heroku)
     - Или использовать только статический фронтенд

2. **Изображения**: 
   - Все изображения должны быть в папке `public/images/`
   - Они автоматически будут доступны после деплоя

3. **Домен**: 
   - Vercel предоставляет бесплатный домен вида `your-project.vercel.app`
   - Можно подключить свой домен в настройках проекта

## Проверка после деплоя

После деплоя проверьте:
- ✅ Сайт открывается
- ✅ Все изображения загружаются
- ✅ Навигация работает
- ✅ Воронка отображается корректно
- ✅ Профиль открывается
