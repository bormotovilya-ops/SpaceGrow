# Проблема с Groq API: 403 Forbidden

## Диагностика

Тестовый запрос показал ошибку **403 Forbidden**. Это означает, что токен неверный или не имеет доступа.

## Возможные причины:

1. **Токен неверный** - возможно, скопирован неправильно
2. **Токен истек** - нужно создать новый
3. **Аккаунт не активирован** - нужно завершить регистрацию на console.groq.com
4. **Токен не имеет прав** - нужно проверить настройки API ключа

## Решение:

### Шаг 1: Проверьте токен на Groq

1. Войдите на https://console.groq.com
2. Перейдите в **API Keys**
3. Проверьте, что токен активен
4. Если токена нет или он неактивен - создайте новый

### Шаг 2: Создайте новый API ключ

1. На https://console.groq.com
2. **API Keys** → **Create API Key**
3. Скопируйте новый ключ (начинается с `gsk_...`)
4. Обновите файл `.env`:
   ```
   GROQ_API_KEY=gsk_новый_токен_здесь
   ```

### Шаг 3: Проверьте аккаунт

- Убедитесь, что регистрация завершена
- Проверьте, что аккаунт активен
- Убедитесь, что вы используете правильный email

### Шаг 4: Проверьте токен

Запустите тест:
```bash
node test-groq.js
```

Если всё правильно, вы увидите успешный ответ.

## Альтернатива: Используйте другую модель

Если проблема с конкретной моделью, можно попробовать другую в `server-local.js`:

```javascript
model: 'llama-3.1-8b-instant', // Вместо llama-3.1-70b-versatile
```

Или:

```javascript
model: 'mixtral-8x7b-32768',
```

## Если проблема сохраняется:

1. Проверьте документацию Groq: https://console.groq.com/docs
2. Убедитесь, что используете правильный endpoint
3. Проверьте лимиты бесплатного тарифа
